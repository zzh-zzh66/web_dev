## 一、项目概述

### 1.1 项目背景

家族族谱系统V1.0已经完成了核心的族谱管理功能，包括用户认证、成员信息管理、族谱树可视化展示等基础能力。为了增强家族成员之间的互动联系，提升用户粘性和使用频率，V2.0版本将引入"个人主页"功能模块，打造一个类似QQ空间的家族社交互动平台。

### 1.2 产品愿景

让每一位家族成员都能拥有属于自己的"数字家族空间"，通过记录生平故事、分享生活点滴、互动交流，将传统族谱从静态的"家族树"升级为动态的"家族社区"，传承家族文化，增进家族凝聚力。

### 1.3 目标用户

| 用户类型 | 特征描述 | 核心诉求 |
|----------|----------|----------|
| 家族长辈 | 50岁以上，关注家族传承 | 记录家族历史、发布家族故事、与晚辈互动 |
| 家族中青年 | 30-50岁，家庭主力 | 管理家庭信息、分享生活动态、关注家族成员 |
| 家族青年 | 18-30岁，社交活跃群体 | 个性化展示、互动交流、关注家族同辈 |
| 家族管理员 | 族谱维护者 | 管理家族内容、审核用户发布、维护秩序 |

### 1.4 核心指标

| 指标 | V1.0现状 | V2.0目标 |
|------|----------|----------|
| 用户日均停留时长 | < 3分钟 | > 10分钟 |
| 用户月活跃率 | 30% | 60% |
| 人均月互动次数 | 0 | > 10次 |
| 内容发布率 | 0% | > 40% |

## 三、需求分析

### 3.1 需求总览

V2.0"个人主页"功能模块包含以下核心功能域：

| 功能域 | 功能点数量 | 优先级 | 说明 |
|--------|------------|--------|------|
| 个人生平内容展示 | 8 | P0 | 时间轴组织的内容发布与展示 |
| 社交互动系统 | 12 | P0 | 留言、评论、点赞、回复体系 |
| 亲属互动机制 | 6 | P0 | 免好友添加的亲属间互动 |
| 用户间互动 | 8 | P1 | 私信、@提醒、话题互动 |
| 响应式布局 | 5 | P0 | PC端+移动端自适应 |
| 示范个人主页 | 3 | P1 | 预设三个典型用户主页示范 |

### 3.2 详细需求分析

#### 3.2.1 个人生平内容展示

**功能描述：**
用户可以创建和管理自己的个人主页，发布包含文字、图片、视频等多媒体内容，系统以时间轴形式组织展示用户的生平记录和动态。

**核心功能点：**

| 编号 | 功能点 | 详细说明 | 优先级 |
|------|--------|----------|--------|
| C-01 | 个人主页创建 | 用户首次访问自动生成个人主页，包含基础信息展示区 | P0 |
| C-02 | 主页自定义 | 可设置主页背景图、个人签名、兴趣标签 | P1 |
| C-03 | 内容发布 | 支持发布文本（富文本）、图片（多图）、视频内容 | P0 |
| C-04 | 内容分类 | 内容可按类型分类：生平日志、家族故事、照片墙、视频集、心情随笔 | P1 |
| C-05 | 时间轴展示 | 内容按发布时间倒序展示，支持按年月筛选查看 | P0 |
| C-06 | 内容编辑/删除 | 作者可编辑或删除自己发布的内容 | P0 |
| C-07 | 内容权限 | 可设置内容可见范围：全家族、仅亲属、仅自己 | P1 |
| C-08 | 内容收藏 | 用户可收藏他人的精彩内容到自己的主页收藏夹 | P2 |

#### 3.2.2 社交互动系统

**功能描述：**
构建完整的社交互动能力，支持用户对内容进行留言、评论、点赞，以及多层级回复功能。

**核心功能点：**

| 编号 | 功能点 | 详细说明 | 优先级 |
|------|--------|----------|--------|
| S-01 | 留言功能 | 可在他人主页留言板发布留言 | P0 |
| S-02 | 评论功能 | 可对任何内容发布评论 | P0 |
| S-03 | 回复功能 | 支持对评论进行多层级回复（最多5层） | P0 |
| S-04 | 点赞功能 | 可对内容、评论、留言进行点赞 | P0 |
| S-05 | 点赞计数显示 | 实时显示点赞数量 | P0 |
| S-06 | 取消操作 | 支持取消点赞、删除自己的评论/留言/回复 | P0 |
| S-07 | @提醒 | 在评论/回复中使用@提及他人，被@者收到通知 | P1 |
| S-08 | 互动通知 | 收到评论、留言、点赞时推送通知 | P0 |
| S-09 | 评论排序 | 支持按时间/热度排序评论 | P1 |
| S-10 | 表情支持 | 评论内容支持使用表情 | P2 |
| S-11 | 评论折叠 | 过长评论自动折叠，点击展开 | P2 |
| S-12 | 热门评论置顶 | 点赞数高的评论优先展示 | P1 |

#### 3.2.3 亲属间免好友添加互动机制

**功能描述：**
利用族谱系统的亲属关系数据，同一族谱内的成员无需添加好友即可直接进行互动，降低互动门槛。

**核心功能点：**

| 编号 | 功能点 | 详细说明 | 优先级 |
|------|--------|----------|--------|
| R-01 | 亲属关系自动识别 | 系统根据族谱关系自动判定亲属身份 | P0 |
| R-02 | 亲属可见性 | 亲属的主页和内容对同族成员默认可见 | P0 |
| R-03 | 免认证互动 | 亲属间留言、评论、点赞无需额外授权 | P0 |
| R-04 | 关系标签展示 | 互动时展示双方关系标签（如：叔侄、兄弟姐妹） | P1 |
| R-05 | 亲属动态优先 | 亲属发布的内容在动态流中优先展示 | P1 |
| R-06 | 跨族谱限制 | 不同族谱用户互动需遵循权限设置 | P0 |

#### 3.2.4 用户间多维度互动

**功能描述：**
提供私信聊天、@提醒、话题标签等多维度互动方式，增强用户间的社交连接。

**核心功能点：**

| 编号 | 功能点 | 详细说明 | 优先级 |
|------|--------|----------|--------|
| M-01 | 私信系统 | 用户间可发送私信消息（文本、图片） | P1 |
| M-02 | 私信列表 | 私信会话列表，按最后消息时间排序 | P1 |
| M-03 | 未读提示 | 未读私信数量提示 | P1 |
| M-04 | @提醒系统 | 评论/回复中@提及用户，生成提醒通知 | P1 |
| M-05 | 通知中心 | 集中展示所有互动通知（评论、点赞、@、私信） | P1 |
| M-06 | 通知分类 | 通知按类型分组：互动通知、系统通知 | P2 |
| M-07 | 话题标签 | 内容发布支持添加#话题#标签 | P2 |
| M-08 | 话题聚合页 | 按话题聚合展示相关内容 | P2 |

#### 3.2.5 响应式布局

**功能描述：**
个人主页及相关页面需同时支持PC端和移动端访问，提供适配不同屏幕尺寸的流畅体验。

**核心功能点：**

| 编号 | 功能点 | 详细说明 | 优先级 |
|------|--------|----------|--------|
| L-01 | PC端布局 | 宽屏三栏布局：左侧导航+中间内容+右侧互动面板 | P0 |
| L-02 | 平板适配 | 双栏或单栏自适应布局 | P1 |
| L-03 | 移动端布局 | 单栏布局，底部导航栏，手势操作支持 | P0 |
| L-04 | 图片自适应 | 图片根据屏幕尺寸自动缩放 | P0 |
| L-05 | 触摸优化 | 移动端按钮大小、间距适合手指操作 | P0 |

#### 3.2.6 示范个人主页

**功能描述：**
创建三个不同风格的示范个人主页，用于展示系统功能和引导用户创建自己的主页。

**示范主页设计：**

| 编号 | 示范主页 | 角色设定 | 内容类型 | 展示重点 |
|------|----------|----------|----------|----------|
| D-01 | 张德厚（族长） | 75岁家族长辈 | 家族历史、祖训家规、老照片 | 厚重的家族传承感，时间轴展示家族重大事件 |
| D-02 | 张建国（中年） | 45岁家族中坚 | 家庭生活、子女成长、工作成就 | 温馨的家庭生活记录，照片墙展示 |
| D-03 | 张晓明（青年） | 25岁年轻一代 | 旅行见闻、学习成长、兴趣爱好 | 活泼的个性化展示，视频和图文混排 |

## 四、用户故事

### 4.1 个人主页相关

#### US-V2-001: 创建个人主页
```
作为 家族成员
我希望 拥有自己的个人主页
以便 展示我的生平故事和生活点滴

验收标准：
- 用户首次访问个人主页时自动生成
- 自动填充来自成员信息的基础资料（姓名、性别、辈分等）
- 可设置个人签名、兴趣标签、主页背景图
- 主页URL可通过链接分享给其他家族成员
```

#### US-V2-002: 发布生平内容
```
作为 家族成员
我希望 在我的主页发布内容（文字、图片、视频）
以便 记录重要的生活时刻和家族故事

验收标准：
- 支持发布纯文本内容（富文本编辑）
- 支持上传多张图片（最多9张），自动压缩
- 支持上传视频（最大500MB），支持mp4格式
- 可选择内容分类：生平日志、家族故事、照片墙、视频集、心情随笔
- 发布后内容立即在个人主页时间轴展示
- 支持设置内容可见范围：全家族、仅亲属、仅自己
```

#### US-V2-003: 浏览他人主页
```
作为 家族成员
我希望 浏览其他家族成员的个人主页
以便 了解他们的生活状态和家族故事

验收标准：
- 可通过成员列表、族谱树节点、@提醒等入口进入他人主页
- 主页展示用户基础信息、生平内容时间轴、留言板
- 只能查看权限允许的内容（根据内容可见范围设置）
- 可进行互动操作：留言、评论、点赞
```

### 4.2 社交互动相关

#### US-V2-004: 留言板互动
```
作为 家族成员
我希望 在他人的主页留言板留言
以便 表达关心和问候

验收标准：
- 在他人主页底部可看到留言板区域
- 可输入文字留言并发布
- 留言按时间倒序排列，最新留言在顶部
- 留言显示留言人头像、姓名、关系标签、时间
- 可删除自己发布的留言
```

#### US-V2-005: 评论与回复
```
作为 家族成员
我希望 对内容进行评论和回复
以便 与他人展开讨论和交流

验收标准：
- 每条内容下方有评论区
- 可发布评论，支持@提醒他人
- 可对评论进行回复（最多5层嵌套）
- 评论和回复显示点赞数
- 支持按时间或热度排序评论
- 可删除自己的评论和回复
```

#### US-V2-006: 点赞互动
```
作为 家族成员
我希望 对内容、评论、留言进行点赞
以便 表达我的认同和喜爱

验收标准：
- 内容、评论、留言旁边有赞按钮
- 点击点赞，按钮高亮，计数+1
- 再次点击取消点赞，计数-1
- 显示总点赞数
- 可看到点赞用户列表（最多显示前10个）
```

### 4.3 亲属互动相关

#### US-V2-007: 亲属间免认证互动
```
作为 家族成员
我希望 无需添加好友即可与族谱中的亲属互动
以便 降低互动门槛，增进家族联系

验收标准：
- 同族谱成员自动识别为亲属关系
- 亲属主页默认对同族成员可见
- 可对亲属的内容直接进行留言、评论、点赞
- 互动时显示双方关系标签（如：伯父、堂兄、表妹等）
- 不同族谱成员的互动遵循内容可见范围设置
```

#### US-V2-008: 查看亲属动态
```
作为 家族成员
我希望 优先看到亲属发布的动态内容
以便 关注家族成员的最新状态

验收标准：
- 动态流中亲属发布的内容优先展示
- 亲属关系越近（代数差越小）的内容越靠前
- 可在动态流中筛选只看亲属动态
```

### 4.4 多维度互动相关

#### US-V2-009: 私信聊天
```
作为 家族成员
我希望 与其他家族成员进行私信聊天
以便 进行私密的沟通交流

验收标准：
- 可从他人主页发起私信
- 支持发送文字和图片消息
- 私信会话按最后消息时间排序
- 未读私信有数量提示
- 仅同族谱成员可发送私信
```

#### US-V2-010: 通知中心
```
作为 家族成员
我希望 在通知中心查看所有互动提醒
以便 及时了解他人的互动反馈

验收标准：
- 通知中心集中展示所有类型的通知
- 通知类型：评论、留言、点赞、@提醒、私信
- 通知按时间倒序排列
- 未读通知有数量角标提示
- 点击通知可跳转到相关内容
- 可一键标记所有通知为已读
```

## 十一、风险与应对

### 11.1 技术风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 多层级评论性能问题 | 查询慢、体验差 | 中 | 采用物化路径或闭包表优化查询 |
| 大文件上传超时 | 视频上传失败 | 中 | 分片上传、断点续传 |
| 通知推送延迟 | 用户体验差 | 低 | 引入WebSocket实时推送 |
| 亲属关系查询复杂 | 关系判定慢 | 中 | 预计算亲属关系、缓存优化 |

### 11.2 项目风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 需求变更 | 延期 | 中 | 严格控制需求范围，P2功能可延后 |
| 联调问题 | 前后端对接困难 | 中 | 先定义接口契约，并行开发 |
| 人员协调 | 专家产出不同步 | 低 | 每日站会同步进度、定期文档评审 |

### 11.3 质量风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 移动端适配不足 | 手机端体验差 | 中 | 早期开始响应式测试 |
| 安全漏洞 | 数据泄露 | 低 | 严格权限控制、输入校验 |
| 数据一致性问题 | 互动数据丢失 | 低 | 事务管理、数据备份 |

## 十二、验收标准

### 12.1 功能验收标准

| 功能域 | 验收标准 |
|--------|----------|
| 个人主页 | 用户可创建、编辑个人主页，展示基础信息和内容 |
| 内容发布 | 支持文本、图片(最多9张)、视频发布，内容可编辑删除 |
| 时间轴展示 | 内容按时间倒序展示，支持分页加载 |
| 评论系统 | 支持评论、多层级回复(5层)、点赞、删除 |
| 留言系统 | 支持在他人主页留言、查看、删除 |
| 点赞系统 | 支持对内容、评论、留言点赞和取消 |
| 通知中心 | 整合所有互动通知，支持未读标记、跳转 |
| 亲属互动 | 同族谱成员自动识别、免认证互动、关系标签展示 |
| 私信功能 | 支持文字消息收发、未读计数 |
| 响应式布局 | PC端三栏布局、移动端单栏布局、平板适配 |

### 12.2 性能验收标准

| 指标 | 要求 |
|------|------|
| 个人主页加载时间 | ≤ 1.5秒（首屏） |
| 内容列表加载 | ≤ 500ms（单页20条） |
| 评论加载 | ≤ 300ms（单层评论） |
| 图片上传 | ≤ 3秒（10MB以内） |
| 视频上传 | 支持断点续传，进度可见 |
| 并发用户数 | 支持1000+同时在线 |

### 12.3 质量验收标准

| 指标 | 要求 |
|------|------|
| 代码覆盖率 | ≥ 70% |
| API接口可用性 | ≥ 99.9% |
| 浏览器兼容性 | Chrome/Firefox/Safari/Edge最新版本 |
| 移动端兼容 | iOS 13+、Android 8+ |
| 安全测试 | 通过OWASP Top 10安全检查 |
