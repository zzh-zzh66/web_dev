## 一、版本概述

V2.0.0 是家族族谱系统的一次重大版本更新,在原有的族谱管理基础功能之上,新增了个人主页功能模块,为家族成员提供了更加丰富的个人展示和互动平台。

本次更新包含5大功能模块、40+个新API接口、14张新数据库表,实现了从单纯的族谱管理工具到家族社交平台的升级。

## 二、新功能列表

### 2.1 个人主页功能模块

#### 个人生平记录与兴趣内容展示
- 用户可创建和编辑个人资料页面
- 支持个人生平经历的详细记录
- 支持兴趣爱好的展示和管理
- 个人资料支持富文本编辑

#### 时间轴组织内容
- 按时间顺序组织个人经历和事件
- 支持自定义时间线节点
- 时间线视图支持筛选和搜索
- 支持按类别分类展示 (生活、工作、旅行等)

#### 多媒体内容支持
- 文本内容: 支持富文本编辑和展示
- 图片内容: 支持图片上传、展示和浏览
- 视频内容: 支持视频链接嵌入和展示
- 图片网格布局,支持点击放大查看
- 支持多种文件格式上传

### 2.2 社交互动系统

#### 多层级评论回复
- 支持对时间轴内容进行评论
- 支持评论的嵌套回复 (无限层级)
- 评论支持富文本内容
- 评论可按时间排序
- 支持评论的编辑和删除

#### 点赞计数与取消
- 支持对时间轴内容进行点赞
- 实时显示点赞数量
- 支持取消点赞操作
- 点赞状态持久化存储
- 防止重复点赞

#### 留言板管理
- 支持在个人主页留言
- 留言支持富文本内容
- 留言可按时间排序浏览
- 支持留言的回复功能
- 留言管理 (编辑、删除)

### 2.3 亲属互动机制

#### 免好友添加直接互动
- 基于亲属关系,无需添加好友即可互动
- 亲属关系自动授予相应的互动权限
- 简化互动流程,提升用户体验

#### 基于亲属关系的权限控制
- 不同亲属关系拥有不同的访问权限
- 隐私设置可自定义内容可见范围
- 支持公开、仅亲属、仅自己三种可见级别
- 权限控制覆盖所有内容类型

### 2.4 用户间互动

#### 私信聊天
- 支持用户间一对一私信聊天
- 私信支持文本内容
- 私信会话列表管理
- 未读消息计数和提醒
- 消息时间线展示

#### @提醒功能
- 支持在评论和留言中@其他用户
- @用户会收到通知提醒
- 点击@用户名可跳转到对应用户主页

#### 通知中心
- 统一的通知中心管理所有通知
- 通知类型包括: 评论回复、点赞、@提醒、私信、系统通知
- 未读消息数量实时显示
- 通知支持标记已读
- 通知列表支持分页浏览

#### 兴趣话题
- 支持创建和参与兴趣话题讨论
- 话题支持分类管理
- 话题内可发布帖子
- 支持话题关注和取消关注
- 热门话题推荐

### 2.5 响应式设计

#### PC端三栏布局
- 左侧: 个人信息卡片
- 中间: 内容展示区域
- 右侧: 互动面板 (评论、留言等)

#### 移动端适配
- 响应式布局自动适配手机屏幕
- 触摸友好的交互设计
- 优化的移动端导航

#### 平板端适配
- 适配7-10英寸平板设备
- 横竖屏自适应
- 合理的内容布局

## 三、技术改进

### 3.1 WebSocket实时通知
- 使用WebSocket实现服务端主动推送
- 实时推送评论回复、点赞、@提醒等通知
- 连接状态管理和自动重连机制
- 心跳保活机制

### 3.2 文件上传功能
- 支持图片、视频等文件上传
- 文件上传进度显示
- 文件大小和类型校验
- 图片压缩和缩略图生成
- 文件存储路径管理

### 3.3 权限控制架构
- 基于角色的访问控制 (RBAC)
- JWT Token认证
- 接口级别的权限校验
- 内容级别的可见性控制
- 隐私设置管理

### 3.4 数据库优化
- 新增14张数据表支持新功能
- 优化查询性能,添加必要索引
- 数据表关联关系设计优化
- 支持分页查询

### 3.5 前端架构改进
- 采用Vue 3 + TypeScript + Vite 技术栈
- Pinia状态管理
- 组件化开发
- 响应式设计 (Element Plus)
- 路由懒加载

### 3.6 后端架构改进
- Spring Boot 框架
- MyBatis Plus 数据访问层
- RESTful API 设计
- 统一异常处理
- 统一响应格式

## 六、升级指南

### 6.1 升级前准备

**重要**: 升级前请务必备份现有数据!

```bash
# 备份数据库 (MySQL示例)
mysqldump -u username -p genealogy_db > backup_$(date +%Y%m%d_%H%M%S).sql

# 备份前端构建产物
cp -r frontend/dist/ backup_dist_$(date +%Y%m%d_%H%M%S)/

# 备份后端jar包
cp backend/target/*.jar backup_jar_$(date +%Y%m%d_%H%M%S)/
```

### 6.2 数据库升级

```sql
-- 步骤1: 连接数据库
mysql -u username -p genealogy_db

-- 步骤2: 执行建表脚本
source db/schema-v2.sql;

-- 步骤3: 执行初始化数据脚本
source db/init-data-v2.sql;

-- 步骤4: 验证表是否创建成功
SHOW TABLES LIKE '%profile%';
SHOW TABLES LIKE '%timeline%';
SHOW TABLES LIKE '%comment%';
```

### 6.3 后端部署

```bash
# 步骤1: 更新配置
# 检查 application.yml 中的数据库配置是否正确

# 步骤2: 编译构建
cd backend
mvn clean package -DskipTests

# 步骤3: 启动应用
java -jar target/genealogy-2.0.0.jar

# 步骤4: 验证启动
curl http://localhost:8080/api/health
```

### 6.4 前端部署

```bash
# 步骤1: 安装依赖
cd frontend
npm install

# 步骤2: 构建
npm run build

# 步骤3: 部署到web服务器
# 将 dist/ 目录下的文件部署到nginx或其他web服务器

# 步骤4: 配置nginx (示例)
server {
    listen 80;
    server_name your-domain.com;
    
    location / {
        root /path/to/dist;
        try_files $uri $uri/ /index.html;
    }
    
    location /api/ {
        proxy_pass http://localhost:8080;
    }
    
    location /ws/ {
        proxy_pass http://localhost:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## 九、升级影响评估

### 9.1 向下兼容性

- **数据库**: V1.0.0的数据库结构保持不变,新增表不影响现有数据
- **API**: V1.0.0的API保持不变,新增API不影响现有调用
- **前端**: V2.0.0为全新页面,与V1.0.0页面独立存在

### 9.2 数据迁移

- V1.0.0的数据无需迁移
- 新增表使用默认初始化数据
- 用户需自行完善个人资料

### 9.3 功能影响

- V1.0.0的族谱管理功能不受影响
- 新增个人主页功能为独立模块
- 用户可选择是否使用新功能

## 十、回滚方案

### 10.1 回滚条件

如升级后出现严重问题,可进行回滚:

- 核心功能无法正常使用
- 数据异常且无法快速修复
- 性能严重下降

### 10.2 回滚步骤

```bash
# 步骤1: 停止当前服务
# 停止后端服务
# 停止前端服务

# 步骤2: 恢复数据库 (如执行了建表脚本)
mysql -u username -p genealogy_db
DROP TABLE IF EXISTS user_profile;
DROP TABLE IF EXISTS timeline_post;
-- ... 删除所有新增表

# 步骤3: 恢复后端
# 替换为备份的jar包
java -jar backup_jar_xxx/genealogy-1.0.0.jar

# 步骤4: 恢复前端
# 替换为备份的dist目录
cp backup_dist_xxx/* /path/to/web/root/
```
